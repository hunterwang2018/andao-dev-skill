# grill-me — 想法打磨（无代码库）

## Goal

将用户的模糊想法通过多轮追问打磨清晰。和 `grill-with-docs` 的区别：**无代码库，不保存任何文件，无状态**。

## Workflow

### 第一步 · 问一句话描述

用户说想法。

### 第二步 · 多轮追问

同 `grill-with-docs` 的追问方向，但**不探索代码库**（因为没有）。

| 方向 | 问题示例 |
|------|---------|
| 目标 | "这个功能解决了什么问题？" |
| 用户 | "谁会使用这个功能？" |
| 边界 | "这个功能不包括什么？" |
| 假设 | "你认为 XX 已经存在，对吗？" |
| 优先级 | "第一期必须有什么？" |

### 第三步 · 停下确认

```
想法已打磨清晰。
核心需求：<一句话总结>

要继续进入 propose 产正式提案吗？
1. 继续
2. 继续打磨
3. 不做了
```

## 约束

- 不保存任何本地文件
- 每轮最多 3 个问题
- 至少 2 轮后才能确认
