# handoff — 上下文桥接

## Goal

在会话需要切换时，将当前对话压缩为结构化摘要，供下一个会话引用。

## Workflow

### 第一步 · 确定桥接方向

问用户：要把上下文带到哪？

1. 进入 prototype 会话
2. 进入另一个新会话
3. 从原型会话带回来

### 第二步 · 压缩内容

总结输出包含：

```markdown
# Handoff — <日期>

## 当前状态
- 项目：<repo 路径>
- 活跃 change：<id>（或无）
- 当前阶段：<phase>

## 已决策
- <决策 1>（理由：<简要理由>）
- <决策 2>

## 未决策 / 待确认
- <待确认项 1>
- <待确认项 2>

## 产物引用
- <产物路径 1>
- <产物路径 2>

## 建议技能
- <后续推荐的技能>
```

### 第三步 · 保存

写入系统临时目录 `C:\Users\<user>\AppData\Local\Temp\opencode\handoff-<timestamp>.md`。

输出：
```
✅ Handoff 已保存到 <路径>
新会话中引用此文件即可恢复上下文。
```

## 约束

- 不修改项目文件
- 不删除源路径中的任何内容
- 敏感信息（密钥/密码）先问用户是否要排除
