# prototype — 方案验证

## Goal

当一个设计问题需要跑起来才能回答时，快速编码验证。产出可丢弃的原型 + 结论，不进入版本控制。

## Workflow

### 第一步 · 确认问题

问用户：你想要验证的具体问题是什么？

### 第二步 · 选择模式

根据问题类型选择：

| 问题类型 | 模式 | 说明 |
|---------|------|------|
| 逻辑/状态/业务规则 | 终端 TUI | 一个可运行的终端程序，通过命令行交互验证状态流转 |
| UI/交互/视觉 | Web 路由 | 实现多个激进不同的 UI 变体，通过 URL 参数切换 |

### 第三步 · 编码

- 从第一天就当作可丢弃的来写
- 一个命令即可运行
- 不做持久化、不做错误处理（除非要验证的就是错误处理）
- 跳过测试、lint、类型检查

### 第四步 · 验证 + 捕获结论

运行原型，用户验证。

记录结论：
```markdown
原型结论：
- 验证问题：<原问题>
- 结论：<验证结果>
- 影响：<对设计的启示>
```

### 第五步 · 清理

原型代码不进入版本控制。捕获结论后，用户可：
1. 将结论带回原会话（通过 `handoff`）
2. 直接在这里继续讨论

## 约束

- 不做超出回答问题的实现
- 不进行 lint/测试/类型检查
- 不纳入版本控制
